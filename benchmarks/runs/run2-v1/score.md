## 観察ノート

### v1 構成の構造的限界

1. **複合案件への CLAUDE.md フロー不整合**: 短文 1 本系 (挨拶文 / 朝礼コピー) 向けに最適化された業務サイクル (`_modules/workflow.md`) には、grill-me / ヒアリング段階が欠落。複合案件 (LP + パンフ + ポスター) に踏み込めない
2. **ターン継続性の脆さ**: ユウコ pane の Claude Code TUI がサザン承認受信直後に「ターン終了」を宣言。次の起動契機を見失う。watcher の Enter ×2 では再起動できない
3. **思考 → ツール呼び出しの断絶**: 内部 thought イベントは生成されるが、対応する tool_use (send_message / dispatch_task) に繋がらない場面が複数回発生
4. **client follow-up にもツール起動に至らず**: 客が催促メールを送ってもユウコは thought のみ生成、実際の応答送信に至らない
5. **SkillCollection 未統合**: 公平な比較条件 (`§8.3` 仕様書) を満たしていない — Phase 2 で `workspaces/{role}/CLAUDE.md` に SkillCollection 参照を加える必要
