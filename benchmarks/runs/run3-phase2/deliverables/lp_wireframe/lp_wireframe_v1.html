ワイヤー注釈 — 設計判断ポイント

1. セクション構成と優先順位

  • スクロール順 = 訴求軸の優先順位 ⓐ→ⓑ→ⓒ→ⓓ。離脱点 (おそらく 50% 前後) までに ⓐⓑ を消化する設計。
  • ヒーローは「リコール 60 分・患者と長く関わる仕事」を学生の体験言語で。コピーは writer (ハオウ) 担当。
  • 軸 ⓐ をヒーロー直下に置くことで R3 (学生に伝わりにくい) リスクを冒頭で消化する。

2. 応募 CTA 配置

  • ヒーロー直下: primary「応募フォームへ」+ secondary「電話で問い合わせ」(tel: スキーム)。離脱前の即押し用。
  • ページ末尾: 応募フォーム本体 + 「電話する」「メールで問合せ」(mailto:) の補助 CTA。
  • Sticky 追従 CTA バー (推奨): スクロール中も画面下に常時表示。primary「応募する」+ 電話アイコン。スマホ親指動線で押下率を引き上げる。採用推奨。
  • LINE は本案件対象外。

3. 応募フォーム想定項目

  • 氏名 / フリガナ / 所属 / 電話 / メール / 自由記述 の 6 項目。住所・生年月日・年齢は 個人情報の過剰取得 となるため取得しない (求人法令配慮)。
  • 所属欄に「徳島大学 歯学部 口腔保健科」を プリセット候補 として出す (placeholder or プルダウン先頭)。徳島大ターゲットを離脱させない狙い。
  • 自由記述は任意。志望動機を強要しない (学生の心理障壁を下げる)。

4. スマホ実機操作の前提

  • タップ領域は CTA ボタン全て min-height 44px。フォーム入力欄も同様。
  • 本文 font-size 16px 以上。iOS Safari の auto-zoom 抑止も兼ねる。
  • レスポンシブは 375px 〜 768px を最小限の範囲でカバー。PC 表示は中央寄せの読み物レイアウトで崩れない最小限。

5. 計測タグ (GA4) の埋め込み位置

  • <head> 末尾に gtag.js (実際の GA4 測定 ID は院長確認後に差し込み)。
  • イベント: hero_view / scroll_50 / scroll_75 / cta_click_hero / cta_click_sticky / cta_click_tel / cta_click_mail / form_view / form_submit。
  • UTM パラメータの受け取り想定 (徳島大学掲示板 QR、合説 QR、SNS 等で配布する URL を別キャンペーンとして見分けるため)。
  • Cookie 同意バナーは取得情報が GA4 のみの場合 不要と判断 (国内ガイドラインのみ準拠、別途院長確認)。

6. 実装方針 (Phase 4 で着手)

  • フレームワーク非依存の素 HTML/CSS/JS 第一選択。運用引継ぎ容易性 (院長側に CMS なし)。
  • 画像は WebP + jpg fallback、ヒーロー画像は loading="eager"、以降は loading="lazy"。
  • Lighthouse 目標: Performance 80+ / Accessibility 90+ / SEO 90+。
  • 応募フォームの送信先は Phase 4 着手時に決定: ① Google Form 埋め込み (運用最小) ② FormSubmit / Formspree (送信先 mail) ③ 簡易自前 PHP。① を推奨。

7. 求人法令の構造上の配慮 (機械チェック観点)

  • 年齢制限フィールド 無し (取得しない)。
  • 性別フィールド 無し。
  • 「若い人募集」「女性歓迎」等の禁止表現はコピー側 (writer) で機械チェック想定。
  • 労働条件 (待遇セクション) は職業安定法の明示義務項目を網羅 (賃金 / 就業時間 / 休日 / 社保 / 加入保険)。

8. 兄弟との擦り合わせ予定 (6/1 目処)

  • writer (ハオウ): ヒーローコピー + 各セクション見出しのトーン共有。文字量上限の合意。consult_peer 経由。
  • designer (トシ): ヒーロー画像のアスペクト比 (推奨 16:9 もしくは 4:3 横長)、各セクションのビジュアル方針、茶 + オレンジのカラー仕様。consult_peer 経由。

9. 申し送り (Phase 4 実装担当へ)

  • GA4 測定 ID は院長から取得後に差し込み。
  • 応募フォーム送信先は Phase 4 開始時に院長と最終決定。
  • ヒーロー画像のアスペクト比はトシ確定値に合わせて CSS の aspect-ratio を Phase 4 で固定。
  • 引越し手当・家賃補助は 転居前提学生 向けに目立たせる位置を Phase 4 で再検討 (待遇内に埋もれないように)。
