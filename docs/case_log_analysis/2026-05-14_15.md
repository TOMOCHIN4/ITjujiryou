## 2. 観点別所見

### 2.1 ★ 権限関連 — 残債と整理

5/14 SUMMARY.md §影響範囲で「Tool(//abs/**) 形式を使っている箇所」が 5 settings.json 全てに残っている問題が指摘済。auto モード切替で **実害は消えた** が、概念的残債としては以下が残る:

| settings.json | 残債 glob | 5/15 での実害 | 対応推奨 |
|---|---|---|---|
| souther | (既に bare Write へ倒し済) | — | (済) |
| yuko | Write/Edit `//.../outputs/**`, `//.../data/memory/yuko/**`, `//.../data/memory/company/_proposals/**` | なし (auto で許可) | **後回し可** — 規律ガード (workflow.md + persona_guard) で代替担保。auto モード前提なら glob を変える緊急性は低い |
| writer | Write/Edit `//.../outputs/**`, `//.../data/memory/writer/**` | なし | 同上 |
| designer | Write/Edit `//.../outputs/**`, `//.../data/memory/designer/**`, **`Bash(//.../scripts/gen-asset/**)`** | なし (5/15 は SVG ルート、gen-asset 起動せず) | **要対応**: 次回 gen-asset 案件が来た時に再発する。`Bash(cmd:*)` prefix glob 化 (例: `Bash(/Users/tomohiro/Desktop/ClaudeCode/ITjujiryou/scripts/gen-asset/venv/bin/python:*)`) で限定化、または bare `Bash` + 規律ガード |
| engineer | Write/Edit `//.../outputs/**`, `//.../data/memory/engineer/**` | なし | 後回し可 |

**所見**: auto モードによる「実害消失」を理由に放置すると、次に dontAsk へ戻る理由が出た時 (例: classifier 不安定発覚 / 公式 path glob fix) に同じ問題が再燃する。**「auto モード前提の運用」を SPEC.md に明文化** して、glob 残債は意図的に放置すると記録するのがクリーン。

**特筆**: 5/15 のユウコは approval_request 本文に **「【リスク】designer の gen-image 環境ブロッカー (既知の Bash 許可ルール glob 問題) 未解決のため、写実画像が出せない場合は代替提案で再上申いたします。」** とリスクを明示している (pane_0 line 35-39)。これは秘書として正しい完遂判断 = 既知の残債を業務上で迂回するスキルが確立されている。
