# print-collateral-build

パンフレット (4P / 8P) およびポスター (A2 / B1 / B2) などの印刷物を制作する WF。

## 適用範囲

- 二つ折り or A4 4P / 8P 程度のパンフレット
- A2 〜 B1 サイズのポスター
- A4 1 枚モノのフライヤー

## 入力

- `initial_plan.md` (ターゲット / 訴求軸 / トーン / 予算 / 納期)
- 配布・掲出経路 (大学掲示板 / 合説 / DM 等)
- ブランドカラー指定
- LP がある場合: LP のコピー・カラーを継承元として参照

## 出力

- パンフレット: 各ページのレイアウト指示 + 文章 + 画像素材を含む Markdown + 画像ファイル
- ポスター: 1 枚のレイアウト指示 + 文章 + 画像素材
- 印刷会社向け入稿前データ (PDF or 高解像度 PNG。後述「穴」参照)
- 申し送り (印刷入稿時の注意 / 修正フロー / 配布想定での視認性チェック結果)

## マクロフロー

### ノード 1: 用途と判型の確定 (ユウコ + トシ + ハオウ)

- アクション:
  1. クライアントから配布・掲出経路をヒアリング済み (initial_plan に含まれる)
  2. パンフレット = A4 二つ折り (4P) を標準とし、情報量が多ければ 8P に拡張
  3. ポスター = 大学掲示板用は A2 (掲示板サイズ規定確認推奨を申し送り)
  4. フライヤーは追加要望時のみ
- 出力: 判型と部数の確定 MD

### ノード 2: 情報設計 (ハオウ + ユウコ)

- アクション:
  1. Skill `marketingskills/marketing-psychology` で「掲示板でチラ見する学生の認知ルート」を整理
  2. パンフレット: 各ページの担い (P1=表紙キャッチ / P2-3=訴求軸 4 本展開 / P4=応募方法 + 連絡先)
  3. ポスター: ファーストインプレッション 3 秒で訴求軸 1 本を伝える設計。詳細は QR コードで LP に誘導
- 出力: 情報設計 MD

### ノード 3: コピーライティング (ハオウ、サブ WF `recruit-copywriting`)

- 起動契機: ノード 2 完了
- アクション:
  1. サブ WF `recruit-copywriting` を起動
  2. パンフレット用: 表紙キャッチ 20 字以内 / 各ページのリード 60 字以内 / 詳細本文 200 字以内 / 応募方法 簡潔に
  3. ポスター用: メインキャッチ 15 字以内 / サブキャッチ 30 字以内 / QR 誘導文 20 字以内
- 出力: 全コピー文言を `outputs/{task_id}/print_copy.md` に集約

### ノード 4: デザイン (トシ主担当)

- 起動契機: ノード 3 完了
- アクション:
  1. Skill `marketingskills/image` で必要なグラフィック素材を生成 (院内雰囲気イラスト / アイコン / 装飾要素)
  2. パンフ・ポスターのレイアウトを Markdown + 画像 placeholder で設計
  3. 色設計: 茶色基調 + 暖色アクセント (LP がある場合は LP 継承)
  4. フォント: 本文 = 明朝系または角ゴシック中明朝、見出し = ゴシック太字
  5. 視認性: ポスターは 2m 離れて視認可能、パンフは 30cm で読みやすい級数

#### 「穴」: 印刷入稿仕様の取り扱い

SkillCollection に CMYK / トンボ / 解像度・出血の入稿仕様 skill は **存在しない**。本 WF では:

- 制作データは **RGB / 72dpi 〜 150dpi** で納品 (画面上の見え方を確認できるレベル)
- 高解像度 (300dpi 印刷用) は **印刷会社のオペレーターレビュー + データ変換が必要**
- トンボ・塗り足しは PDF 出力時に印刷会社が付与する想定
- これらを **申し送り書に明記** することで「印刷会社経由で実運用可能」状態を担保

- 出力: パンフ・ポスターのレイアウト指示 + 画像素材一式

### ノード 5: コピー校正 (ハオウ)

- 起動契機: ノード 4 完了
- アクション:
  1. Skill `marketingskills/copy-editing` を実行
  2. 誤字脱字 / 文字数制約遵守 / 求人法令上の禁則 / 転居前提との整合性をチェック
- 出力: 校正済みコピー

### ノード 6: デザインレビュー (トシ + ユウコ)

- 起動契機: ノード 5 完了
- アクション:
  1. Skill `skills/design-review` を実行
  2. パンフレット展開図 / ポスター視認性チェック (掲示板距離想定)
  3. QR コードのテスト読み取り (ポスター)
- 出力: レビュー結果 MD

### ノード 7: ユウコ統合バリデーション

- 起動契機: ノード 6 完了
- アクション:
  1. パンフレット・ポスターの整合性 (用語 / 色 / 訴求軸の優先順位)
  2. ペルソナ漏れチェック
  3. 求人法令上の禁止表現チェック
  4. 印刷入稿時の留意点を申し送りに明記
- 出力: バリデーション結果 + 印刷物完成版

## 申し送り書 (本 WF が常に生成する)

- パンフレット・ポスターの仕様 (判型 / 用紙 / 想定印刷部数)
- 各ページの構成意図
- ブランドカラーの使用箇所 (色コード一覧)
- **印刷会社への入稿前に必要な作業** (CMYK 変換 / トンボ・塗り足し付与 / 解像度確認)
- QR コードの遷移先 URL と動作確認結果
- 修正フロー (テキスト微修正は折返し 1 営業日 / レイアウト変更は 3 営業日 / 印刷直前修正は不可)

## 「穴」の取り扱い (再掲)

- **印刷入稿仕様**: CMYK / トンボ / 出血の付与は **印刷会社オペレーターレビュー必須**
- **校了印**: 印刷物の最終校了は **クライアント (院長) のサイン責任** で、agency 側は素材と仕様提案までを範囲とする

## 案件アレンジ時の改変ガイド

- 印刷物の種類が増減する場合は frontmatter の `agents` 直下に `case-collateral-types: [pamphlet-4P, poster-A2, flyer-A4]` を追加して明示
- ノード 1 で判型が確定したら、後続ノードの「ポスター」または「パンフレット」のフローを片方残す
