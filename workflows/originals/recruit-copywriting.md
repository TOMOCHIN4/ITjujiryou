# recruit-copywriting

求人案件におけるコピーライティング専用 WF。LP / パンフ / ポスター それぞれの WF からサブとして呼ばれる、または単発のコピー案件で単独実行される。

## 適用範囲

- 求人媒体のキャッチコピー (1〜3 案)
- 求人 LP / パンフ / ポスター用のコピー一式
- 求人広告クリエイティブ (Instagram / X / Facebook 用 ad text)

## 入力

- ターゲット読者プロファイル (例: 徳島大歯学部口腔保健科 3-4 年生、徳島実家、大阪転居前提)
- 訴求軸の優先順位 (4 本以内)
- 文字数制約 (一覧表形式)
- トーン&マナー (フォーマル度、敬語 / 半敬語 / カジュアル)
- 既存ブランドの口調があれば参考に

## 出力

- コピー一式 (キャッチコピー、サブコピー、リード文、本文、CTA 文言)
- 採用された案と没案の理由メモ
- 求人法令上の留意点リスト

## マクロフロー

### ノード 1: ターゲット深堀り (ハオウ)

- アクション:
  1. Skill `marketingskills/customer-research` を実行
  2. ターゲット読者の生活実態 / 進路意識 / 不安 / 期待を具体化
  3. ペルソナ 1 体を「徳島大歯学部口腔保健科 4 年・原田美咲・徳島市出身・実家暮らし・進路未定・親は地元就職を期待」のように肉付け (架空でも具体的に)
- 出力: ターゲットペルソナ MD

### ノード 2: 心理障壁の抽出 (ハオウ)

- アクション:
  1. Skill `marketingskills/marketing-psychology` を実行
  2. 「徳島から大阪へ転居する」決断を阻む心理障壁を 5〜7 個列挙:
     - 親の反対 / 寂しさ / 慣れない大阪での生活費 / 言葉の違い / 友人関係の希薄化 / 仕事への不安 / 家賃補助があっても初期費用が心配
  3. それぞれに対する **当院の回答** を整理 (家賃半額補助 + 引越し手当 / リコール 60 分で学べる環境 / 院長が説明好き / スタッフ仲良し)
- 出力: 心理障壁マップ MD

### ノード 3: 訴求軸の言語化 (ハオウ)

- アクション:
  1. 訴求軸 4 本 (リコール 60 分 / 個室 9 台 / 子育て両立 / 院長から学べる) を、ターゲット読者の「ことば」に翻訳
  2. 例: 「リコール 60 分」→「患者さんと、ちゃんと向き合える時間がある」
  3. 各訴求軸につき 3〜5 個の言い換え候補を用意
- 出力: 訴求軸辞書 MD

### ノード 4: キャッチコピー生成 (ハオウ、Skill `copywriting`)

- アクション:
  1. Skill `marketingskills/copywriting` を実行
  2. メインキャッチを 3 案生成 (各 15〜20 字以内)
  3. サブコピーを 3 案生成 (各 30〜40 字以内)
  4. 各案について「狙い」「想定読者の反応」「リスク」を 1 行ずつ添える
- 出力: キャッチコピー 3 案 + 解説

### ノード 5: 広告クリエイティブ系 (必要な場合のみ、Skill `ad-creative`)

- 起動契機: 広告配信を含む案件のみ
- アクション:
  1. Skill `marketingskills/ad-creative` を実行
  2. Instagram / X / Facebook 用の ad text を生成 (各媒体の文字数制約遵守)
- 出力: 広告クリエイティブ MD

### ノード 6: 本文・リード生成 (ハオウ)

- アクション:
  1. キャッチコピーの選定後、LP / パンフ / ポスターの各セクションのリード文 + 本文を生成
  2. 文字数制約を厳守
  3. ターゲットの「ことば」に寄せた書き方 (フォーマルすぎず、馴れ馴れしすぎず)
- 出力: 全文コピー MD

### ノード 7: 校正と法令チェック (ハオウ、Skill `copy-editing`)

- アクション:
  1. Skill `marketingskills/copy-editing` を実行
  2. 誤字脱字 / 文字数 / 用語統一を機械的にチェック
  3. **求人法令上の禁則** を手動チェック (下記参照)

#### 求人法令チェックリスト ("穴" — 専門家レビュー推奨)

SkillCollection には求人法令専用 skill がないため、本 WF では以下の **基本チェック** のみを行い、専門家レビューを申し送りに明記する:

- 性別による限定 (例: 「女性歯科衛生士」「女性が活躍中」のうち、後者は OK、前者は NG)
- 年齢制限の明記 (「○歳まで」は職業安定法上原則 NG、例外規定の理由が必要)
- 国籍・身体的特徴による限定 (NG)
- 「アットホームな職場」など空虚な訴求のリスク (法令違反ではないが応募者の反発を招く)
- 給与額の最低保証額の表示 (固定給最低額を明示)
- 試用期間の条件明記 (試用期間中の給与が異なる場合は明示)

**重要**: 本 WF は最終チェックを保証しない。クライアント側で社労士・弁護士レビューを推奨する旨を申し送りに明記。

- 出力: 校正済みコピー + 法令留意点リスト

### ノード 8: 没案の理由メモ

- アクション:
  1. 没にした案について理由を 1 行で書き残す (self/memory に追記。将来同案件で「同じ理由で同じ過ちを繰り返さない」材料に)
- 出力: 没案メモ (self/memory 内)

## 申し送り書 (本 WF が常に生成する)

- 採用コピー一覧 (キャッチ / サブ / リード / 本文 / CTA)
- 没案と理由
- 訴求軸辞書 (用語ゆれを防ぐためにクライアントに共有)
- 求人法令上の留意点リスト + **専門家レビュー推奨** の明記
- 修正フロー (テキスト微修正は即日対応 / コピー方針の変更は 2 営業日)

## 「穴」の取り扱い

- **求人法令の詳細**: 専門家レビューを推奨。本 WF では基本チェックのみ
- **多言語対応**: 範囲外
- **動画スクリプト**: 範囲外。別案件で

## 案件アレンジ時の改変ガイド

- 文字数制約が案件で大きく変わる場合は frontmatter に `case-char-limits: {catch: 20, sub: 40, lead: 80, body: 200}` を追記
- 訴求軸の数が 3 本 or 5 本の案件では、ノード 3 を該当本数で調整
